# OpenClaw 5.x Upgrade Findings (2026-05-07)

> **TL;DR**: Don't upgrade openclaw past **2026.4.27** with the current `@larksuite/openclaw-lark` plugin. The OpenClaw 5.x line introduced strict plugin contracts that the Lark-team plugin hasn't adopted yet (capped at 2026.4.10). Switching to the alternative `@openclaw/feishu` plugin is *possible* on 5.x but is a non-trivial migration that loses some Feishu tool surface.

## Context

In the course of the 2026-05-07 sidecar pidfile work (PRs #21–#23), we tried to upgrade prod openclaw from `2026.4.1` → `2026.5.6` (latest at the time). Several breaking changes surfaced. This note records what broke, what we learned, and the path forward.

## Environment under test

- **Prod**: `/opt/homebrew/lib/node_modules/openclaw` (npm-installed global), launchd-managed gateway + sidecar + channel-server.
- **Sidecar plugin** (`openclaw-sidecar-plugin/` in this repo): registers `before_dispatch` hook + 4 slash commands (`/status /agents /logs /broadcast`).
- **Channel plugin**: `@larksuite/openclaw-lark@2026.4.10` (latest from Lark team).

## Breaking changes hit

### 1. `package.json` must declare `openclaw.extensions`

5.x rejects plugins missing this field:

```
package.json missing openclaw.extensions; update the plugin package to include
openclaw.extensions (for example ["./dist/index.js"])
```

Our sidecar plugin's `package.json` was 4.x-format. Fixed in PR #23 (`openclaw.extensions: ["./index.js"]`). The change is purely additive metadata — 4.x ignores it, so safe to ship before any upgrade.

### 2. `/status` is reserved as a built-in command in 5.x

When our sidecar plugin runs under 5.6:

```
[plugins] command registration failed: Command name "status" is reserved by a
built-in command (plugin=openclaw-sidecar)
```

The plugin's `register()` proceeds — only the `/status` command fails. The other three (`/agents /logs /broadcast`) appear to register OK (no errors logged for them). To use `/status` in 5.x we'd need to rename it to e.g. `/sidecar-status` or override the built-in.

### 3. Plugins registering agent tools must declare `contracts.tools`

```
[plugins] plugin must declare contracts.tools before registering agent tools
(plugin=openclaw-lark, source=/Users/h2oslabs/.openclaw/extensions/openclaw-lark/index.js)
```

Repeated 39× on every gateway boot. The Lark-team plugin has not adopted this contract. **This is the showstopper** — Feishu agent tools (`feishu_chat`, `feishu_im_*`, `feishu_calendar_*`, etc.) silently fail to register, leaving the bot without tool-calling capability for those domains.

### 4. Config schema validation tightened

A pre-existing `agents.list[15].tools.allow + alsoAllow` conflict (caused earlier today by a fragmented lark plugin reinstall) blocked 5.x's stricter validator with:

```
agents.list.15.tools: agent tools cannot set both allow and alsoAllow in the
same scope (merge alsoAllow into allow, or remove allow and use profile + alsoAllow)
```

4.x had been silently tolerating the conflict. 5.x refuses to start any plugin install command until it's resolved. We fixed manually (merged `allow=['exec']` into `alsoAllow`).

### 5. `openclaw plugins install` no longer supports `--force`

4.x: `openclaw plugins install file:./plugin --force` (replace existing install)
5.x: removed; now uses `--dangerously-force-unsafe-install` for a different purpose.

To re-deploy a plugin in 5.x: `rm -rf ~/.openclaw/extensions/<name>` then `openclaw plugins install file:...`.

## What survived

Despite the lark contract issue, these did keep working under 5.6:

- **Sidecar Python service** (`/api/v1/agents` returns HTTP 200) — completely independent of openclaw version.
- **Our sidecar plugin's `register()` callback** — runs successfully, prints `Sidecar plugin registered (url=http://127.0.0.1:18791, account=shared)` in gateway log.
- **Plugin URL discovery** (PR #22) — pidfile path resolution works (`/Users/h2oslabs/.openclaw/sidecar.pid`).

## What we couldn't observe

- **`before_dispatch` hook firing under 5.6** — no fallback-routed Feishu messages came in during the ~10-min 5.6 window. All received messages routed to existing provisioned agents, so the hook condition (`sessionKey.includes("fallback")`) never tripped. Cannot confirm whether the hook contract is unchanged in 5.x without deliberately sending a message from an unprovisioned user.
- **`event` / `ctx` shape under 5.6** — same reason; no fallback dispatch happened.

## Two plugins, one Feishu

There are **two distinct Feishu plugins** for openclaw — discovered while debugging:

| | `@larksuite/openclaw-lark` (current prod) | `@openclaw/feishu` (alternative) |
| --- | --- | --- |
| Maintainer | Lark team | openclaw community (`@m1heng`, `steipete`, `vincentkoc`) |
| Latest | `2026.4.10` | `2026.5.6` (current with openclaw) |
| Channel id | `openclaw-lark` | `feishu` |
| Plugin id | `openclaw-lark` | `@openclaw/feishu` |
| Peer dep | `openclaw: >=2026.3.22` | `openclaw: >=2026.5.6` |
| Declares `contracts.tools`? | No | Yes |
| Tool surface | ~37 (`feishu_calendar_*`, `feishu_im_*`, `feishu_oauth*`, `feishu_drive_*`, `feishu_doc_*`, `feishu_bitable_*`, `feishu_search_*`, `feishu_get_user`, …) | 14 (`feishu_chat`, `feishu_doc`, `feishu_drive`, `feishu_perm`, `feishu_wiki`, `feishu_bitable_*`) |
| Setup CLI | `npx @larksuite/openclaw-lark-tools install` (device-code OAuth flow) | (built-in plugin install, manual app config) |

**Switching plugins is not drop-in:**
- Different channel id means `channels.openclaw-lark.*` config keys all need migration to `channels.feishu.*`.
- Tool surface gap: `@openclaw/feishu` lacks `feishu_calendar_*`, `feishu_im_*`, `feishu_oauth*` — any feature relying on those breaks.
- Setup flow is different: lark-tools provides a device-code "one-click" Feishu app registration; `@openclaw/feishu` expects you to register the app yourself.

## Recommendation

**Stay on `@larksuite/openclaw-lark` + openclaw 4.x for the foreseeable future.**

- Maximum compat for the lark plugin: **`openclaw 2026.4.27`** (last 4.x with full lark plugin support, per lark plugin docs).
- Upgrade prod to `2026.4.27` as a small step (4.1 → 4.27 = 26 days of fixes, no major contract changes).
- Defer `@openclaw/feishu` migration evaluation to a separate work item — it's not just a plugin swap, it's a feature audit + config rewrite.
- Watch lark plugin for `2026.5.x` release; once it lands, re-evaluate openclaw 5.x upgrade.
